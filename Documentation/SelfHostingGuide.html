1

Create a VM or LXC Container on Proxmox

1.1
Choose VM or LXC

For a simple web app, an LXC container is recommended — it's lighter and faster to set up than a full VM. Use a VM only if you need a full isolated OS.

In the Proxmox web UI: Create CT → choose Ubuntu 22.04 or Debian 12 as the template → allocate at least 1 CPU core, 1GB RAM, 10GB disk.

1.2
Set a static local IP

During container creation, set a static IP in the Network tab (e.g. 192.168.1.50/24, gateway 192.168.1.1). This keeps things predictable.

1.3
Start and connect

Click the container → Start → Console. Or SSH in from your main machine:

ssh root@192.168.1.50

Run a quick update first:

apt update && apt upgrade -y
2

Install Node.js and MySQL

2.1
Install Node.js (v20 LTS) 
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

Verify: node -v and npm -v

2.2
Install MySQL 
apt install -y mysql-server
systemctl enable --now mysql
mysql_secure_installation
2.3
Create your database and user 
mysql -u root -p
CREATE DATABASE myapp;
CREATE USER 'myapp_user'@'localhost' IDENTIFIED BY 'strongpassword';
GRANT ALL PRIVILEGES ON myapp.* TO 'myapp_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
2.4
Import your existing data

On your local machine, export your database first:

mysqldump -u root -p your_local_db > backup.sql

Then copy it to the server and import it:

scp backup.sql root@192.168.1.50:/root/
ssh root@192.168.1.50
mysql -u myapp_user -p myapp < /root/backup.sql
3

Deploy Your App

3.1
Build your React frontend

On your local machine, run:

npm run build

This generates a dist/ folder of static files.

3.2
Serve the frontend from Express

Add these lines to your Node/Express server so it serves the built React app:

import path from 'path';
import { fileURLToPath } from 'url';
const __dirname = path.dirname(fileURLToPath(import.meta.url));

app.use(express.static(path.join(__dirname, 'dist')));
app.get('/{*path}', (req, res) => {
  res.sendFile(path.join(__dirname, 'dist', 'index.html'));
});
3.3
Use environment variables for DB config 
const db = mysql.createConnection({
  host:     process.env.DB_HOST     || 'localhost',
  user:     process.env.DB_USER     || 'myapp_user',
  password: process.env.DB_PASSWORD || '',
  database: process.env.DB_NAME     || 'myapp',
});

Create a .env file in your project root on the server (and add it to .gitignore):

DB_HOST=localhost
DB_USER=myapp_user
DB_PASSWORD=strongpassword
DB_NAME=myapp
PORT=3000
3.4
Copy your project to the server 
rsync -av --exclude='node_modules' --exclude='.git' \
  ./my-project/ root@192.168.1.50:/var/www/myapp/

Then on the server, install dependencies:

cd /var/www/myapp
npm install --production
3.5
Test it manually 
node server.js

Visit http://192.168.1.50:3000 from your local machine. If it works, press Ctrl+C and move on.

4

Keep It Running with PM2

4.1
Install and start PM2

PM2 keeps your app running in the background and restarts it if it crashes.

npm install -g pm2
cd /var/www/myapp
pm2 start server.js --name myapp
4.2
Make it survive reboots 
pm2 startup
pm2 save
Tip: Use pm2 logs myapp to see output, pm2 restart myapp after code changes, pm2 status to check it's running.
5

Set Up Cloudflare Tunnel

Why Cloudflare Tunnel? Since you don't have router access, this is the cleanest solution. Your server reaches out to Cloudflare — no inbound ports need to be opened. Cloudflare also handles HTTPS automatically.
5.1
Get a domain name

Cloudflare Tunnel requires a domain. Buy one from Porkbun or Namecheap (~$10–15/year). Then go to cloudflare.com, add your domain, and update your nameservers to Cloudflare's.

5.2
Install cloudflared 
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg \
  | tee /usr/share/keyrings/cloudflare-main.gpg > /dev/null

echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] \
  https://pkg.cloudflare.com/cloudflared any main" \
  | tee /etc/apt/sources.list.d/cloudflared.list

apt update && apt install -y cloudflared
5.3
Authenticate and create a tunnel 
cloudflared tunnel login
cloudflared tunnel create myapp
5.4
Configure the tunnel 
mkdir -p ~/.cloudflared
nano ~/.cloudflared/config.yml

Paste this (replace with your tunnel ID and domain):

tunnel: YOUR_TUNNEL_ID_HERE
credentials-file: /root/.cloudflared/YOUR_TUNNEL_ID_HERE.json

ingress:
  - hostname: yourdomain.com
    service: http://localhost:3000
  - service: http_status:404
5.5
Point DNS to the tunnel 
cloudflared tunnel route dns myapp yourdomain.com
5.6
Run tunnel as a system service 
cloudflared service install
systemctl start cloudflared
systemctl enable cloudflared

Your site should now be live at https://yourdomain.com — with HTTPS handled automatically by Cloudflare!

6

Final Checks

6.1
Checklist before sharing the link

✅ No localhost URLs hardcoded in your frontend or backend

✅ .env file is on the server and in .gitignore

✅ pm2 status shows your app as online

✅ systemctl status cloudflared shows active (running)

✅ Visit your domain from a phone on mobile data to confirm it's publicly accessible

6.2
Updating your app later

On your local machine — rebuild and sync:

npm run build
rsync -av --exclude='node_modules' --exclude='.git' \
  ./my-project/ root@192.168.1.50:/var/www/myapp/

Then on the server:

pm2 restart myapp
