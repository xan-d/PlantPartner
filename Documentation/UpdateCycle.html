🌿 Plant Partner — Deploy Cheatsheet

Quick reference for pushing changes to production

Local → WSL → Proxmox Plant Partner Website Squarespace Domain port 9275
🎨

Frontend Change

Modified anything in /frontend/src — components, styles, pages

1
Build 💻 Local machine 
npm run build
2
Sync 🐧 WSL 
rsync -av --exclude='node_modules' --exclude='.git' \
  /mnt/c/Users/xande/OneDrive/Documents/GitHub/PlantPartner/ \
  root@192.168.1.50:/var/www/myapp/
3
Restart 🖥️ Proxmox server 
pm2 restart plantpartner
⚙️

Backend Only Change

Modified server.js, routes, or other backend files — no frontend changes

1
Sync 🐧 WSL 
rsync -av --exclude='node_modules' --exclude='.git' \
  /mnt/c/Users/xande/OneDrive/Documents/GitHub/PlantPartner/ \
  root@192.168.1.50:/var/www/myapp/
2
Restart 🖥️ Proxmox server 
pm2 restart plantpartner
🔄

Useful Server Commands

Handy commands for checking status and debugging on the Proxmox server

▶
Status & Logs 🖥️ Proxmox server 
pm2 status
Check if app is running
pm2 logs plantpartner
View live logs
pm2 logs plantpartner --lines 30
View last 30 lines of logs
systemctl status cloudflared
Check tunnel is running
Remember: if you add new npm packages locally, run npm install on the server after syncing — packages in node_modules are excluded from rsync.
